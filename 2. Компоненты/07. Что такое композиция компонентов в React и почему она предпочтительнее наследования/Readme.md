## Краткая шпаргалка

> Композиция компонентов — это основной способ переиспользования и расширения компонентов в React. Вместо создания новых компонентов через наследование мы комбинируем существующие через `children`, props, slots, compound components и кастомные hooks. Такой подход позволяет расширять поведение компонентов без изменения их внутренней реализации, делая приложение более гибким и масштабируемым.

## Нюансы и выжимка для собеса

* Композиция — основной способ расширения поведения компонентов в React.

* React рекомендует композицию вместо наследования.

* Самый распространенный механизм композиции — `children`.

* Важные паттерны композиции:

  * `children`;
  * slots;
  * compound components;
  * render props (исторически).

* Compound Components активно используются в UI-библиотеках и часто обсуждаются на Middle+/Senior интервью.

* Кастомные Hooks не заменяют композицию, а дополняют её:

  * Hooks переиспользуют логику;
  * композиция переиспользует UI.

* Главное преимущество композиции:

  > Позволяет расширять поведение компонента без изменения его внутренней реализации.

* Композиция хорошо сочетается с принципом Open/Closed Principle (OCP): компонент открыт для расширения и закрыт для модификации.

* Типичная ошибка — создавать множество специализированных компонентов через копирование или наследование вместо настройки универсального компонента через props и композицию.

* На собеседованиях часто спрашивают:

  * композиция vs наследование;
  * `children` vs обычные props;
  * Compound Components;
  * Render Props vs Hooks;
  * способы переиспользования логики в React.

## Своими словами

Композиция — это основной способ построения интерфейсов в React. Вместо того чтобы создавать новые компоненты через наследование, мы собираем их из уже существующих компонентов. Чаще всего это делается через `children`, но также используются slots и Compound Components. Благодаря композиции компонент можно расширять и настраивать, не меняя его внутренний код. Это делает архитектуру более гибкой и упрощает поддержку приложения. В современном React обычно разделяют две задачи: логику переиспользуют через кастомные hooks, а интерфейс собирают через композицию компонентов. Именно поэтому композиция считается фундаментальным паттерном React и используется практически во всех крупных приложениях и UI-библиотеках.

## Основная часть

### Что такое композиция компонентов

Композиция — это подход, при котором сложный интерфейс собирается из более мелких и независимых компонентов.

В React это считается основным механизмом построения приложений. Вместо создания иерархий компонентов через наследование мы комбинируем уже существующие компоненты между собой.

Простейший пример:

```jsx
function Card({ children }) {
  return (
    <div className="card">
      {children}
    </div>
  );
}

function UserProfile() {
  return (
    <Card>
      <h2>Alex</h2>
      <p>Frontend Developer</p>
    </Card>
  );
}
```

Компонент `Card` не знает, что находится внутри него. Он предоставляет только структуру, а конкретное содержимое передается извне.

---

### Зачем нужна композиция

Композиция решает несколько задач:

* переиспользование UI-компонентов;
* снижение связанности между компонентами;
* создание гибких и расширяемых интерфейсов;
* упрощение тестирования и поддержки;
* возможность расширять функциональность без изменения существующего кода.

Ключевая идея:

> Компонент можно расширять через композицию, не изменяя его внутреннюю реализацию.

Например:

```jsx
<Card>
  <UserInfo />
</Card>
```

Позже:

```jsx
<Card>
  <UserInfo />
  <UserActions />
</Card>
```

Сам компонент `Card` менять не пришлось.

---

### Основные способы композиции

#### Через children

Самый распространенный вариант.

```jsx
<Modal>
  <UserForm />
</Modal>
```

`Modal` отвечает только за отображение модального окна, а содержимое приходит через `children`.

---

#### Через props (Slots Pattern)

Иногда компоненты передаются через отдельные props.

```jsx
function Layout({ header, sidebar, content }) {
  return (
    <>
      <header>{header}</header>
      <aside>{sidebar}</aside>
      <main>{content}</main>
    </>
  );
}

<Layout
  header={<Header />}
  sidebar={<Sidebar />}
  content={<Dashboard />}
/>
```

Такой подход часто называют **Slots Pattern**.

Он позволяет гибко конфигурировать структуру компонента.

---

#### Compound Components

Один из самых популярных паттернов в современных UI-библиотеках.

Пример:

```jsx
<Dialog>
  <Dialog.Title />
  <Dialog.Content />
  <Dialog.Actions />
</Dialog>
```

Или:

```jsx
<Tabs>
  <Tabs.List>
    <Tabs.Tab />
    <Tabs.Tab />
  </Tabs.List>

  <Tabs.Panel />
</Tabs>
```

Внешне компонент выглядит как единое целое, но внутри состоит из нескольких связанных компонентов, которые взаимодействуют через общий контекст или внутреннее состояние.

Такой подход используется во многих UI-библиотеках, потому что позволяет создавать выразительный и масштабируемый API.

---

#### Render Props

Исторически одним из способов композиции были Render Props.

```jsx
<DataProvider
  render={(data) => <UserList users={data} />}
/>
```

После появления Hooks этот паттерн используется значительно реже, но его всё ещё можно встретить в старых проектах и библиотеках.

---

### Почему композиция предпочтительнее наследования

В классическом ООП часто используется наследование:

```js
class AdminButton extends Button {}
```

Проблемы такого подхода:

* появляются глубокие иерархии классов;
* растет связанность между сущностями;
* усложняется расширение функциональности;
* становится сложнее поддерживать код.

В React вместо этого обычно используется композиция:

```jsx
<Button icon={<AdminIcon />}>
  Admin Panel
</Button>
```

или

```jsx
<Button variant="admin">
  Admin Panel
</Button>
```

Мы не создаем новый компонент через наследование, а настраиваем существующий через композицию и props.

Именно поэтому в React практически нет реальных сценариев, где наследование является предпочтительным решением.

---

### Связь композиции и Hooks

В современном React важно разделять переиспользование логики и переиспользование интерфейса.

Логика обычно переиспользуется через кастомные hooks:

```jsx
function UsersPage() {
  const { users, loading } = useUsers();

  return (
    <UsersTable
      users={users}
      loading={loading}
    />
  );
}
```

Интерфейс при этом собирается через композицию компонентов.

Поэтому хороший тезис для собеседования:

> Логику переиспользуем через Hooks, UI — через композицию компонентов.

---

### Где композиция используется на практике

Практически в любом React-приложении.

#### Layout-компоненты

```jsx
<PageLayout>
  <Sidebar />
  <Content />
</PageLayout>
```

#### Модальные окна

```jsx
<Modal>
  <CreateUserForm />
</Modal>
```

#### Таблицы

```jsx
<Table
  header={<UsersHeader />}
  body={<UsersBody />}
/>
```

#### UI-библиотеки

```jsx
<Dialog>
  <Dialog.Title />
  <Dialog.Content />
  <Dialog.Actions />
</Dialog>
```

Большинство современных компонентных библиотек строятся именно на композиции.
