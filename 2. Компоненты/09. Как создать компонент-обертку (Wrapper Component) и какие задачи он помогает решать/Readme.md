## Краткая шпаргалка

> Wrapper Component (компонент-обертка) — это компонент, который оборачивает другие компоненты и добавляет им общее поведение, логику или оформление. Обычно такие компоненты работают через `children` и позволяют переиспользовать функциональность без дублирования кода. Wrapper-компоненты часто используются для авторизации, работы с правами доступа, лейаутов, обработки ошибок, логирования и стилизации.

## Нюансы и выжимка для собеса

* Wrapper Component — компонент, который оборачивает другие компоненты через `children`.
* Является одним из основных инструментов композиции в React.
* Позволяет переиспользовать общий UI и поведение без копирования кода.
* Часто используется для:

  * Layout-компонентов;
  * авторизации;
  * проверки прав доступа;
  * провайдеров контекста;
  * обработки ошибок;
  * логирования и аналитики.
* Хорошая обертка должна решать одну конкретную задачу.
* Не стоит превращать Wrapper Component в «универсальный комбайн», который отвечает сразу за всё.
* Wrapper Components хорошо сочетаются с кастомными Hooks:

  * Hooks переиспользуют логику;
  * Wrapper Components переиспользуют структуру и поведение интерфейса.
* Провайдеры (`AuthProvider`, `ThemeProvider`, `QueryClientProvider`) фактически являются компонентами-обертками.
* На собеседованиях любят спрашивать:

  * Wrapper Component vs HOC;
  * Wrapper Component vs Render Props;
  * зачем нужен `children`;
  * как через обертки реализовать защиту роутов или проверку прав доступа.

## Своими словами

Компонент-обертка — это компонент, который принимает другие компоненты через `children` и добавляет им какое-то общее поведение. Вместо того чтобы дублировать один и тот же код на разных страницах, мы выносим его в отдельную обертку и используем повторно. Чаще всего через такие компоненты реализуют лейауты, авторизацию, права доступа, провайдеры контекста и обработку ошибок. Wrapper Component является одним из самых распространенных примеров композиции в React. Он помогает разделять ответственность между компонентами и делает код более поддерживаемым. В современном React часто используется связка: логика находится в кастомных hooks, а управление структурой и поведением интерфейса — в компонентах-обертках. Для собеседования важно понимать, что Wrapper Component — это не отдельная технология React, а архитектурный паттерн, построенный на использовании `children` и композиции компонентов.

## Основная часть

### Что такое Wrapper Component

Компонент-обертка — это компонент, который принимает другие компоненты через `children` и выполняет некоторую дополнительную работу вокруг них.

Простейший пример:

```jsx
function Card({ children }) {
  return (
    <div className="card">
      {children}
    </div>
  );
}
```

Использование:

```jsx
<Card>
  <UserProfile />
</Card>
```

Здесь `Card` не знает, что именно находится внутри. Его задача — предоставить общий внешний вид для вложенного контента.

---

### Как работает Wrapper Component

Основная идея заключается в том, что обертка берет на себя общую функциональность, а бизнес-логика остается во вложенных компонентах.

```jsx
function PageLayout({ children }) {
  return (
    <>
      <Header />
      <main>{children}</main>
      <Footer />
    </>
  );
}
```

Использование:

```jsx
<PageLayout>
  <Dashboard />
</PageLayout>
```

В результате любой экран автоматически получает одинаковую структуру страницы.

---

### Какие задачи решают Wrapper Components

### 1. Переиспользование UI

Очень часто обертка отвечает только за внешний вид.

```jsx
function Section({ children }) {
  return (
    <section className="section">
      {children}
    </section>
  );
}
```

Использование:

```jsx
<Section>
  <Products />
</Section>
```

Позволяет централизованно управлять оформлением.

---

### 2. Layout-компоненты

Один из самых популярных кейсов.

```jsx
function DashboardLayout({ children }) {
  return (
    <>
      <Sidebar />
      <main>{children}</main>
    </>
  );
}
```

```jsx
<DashboardLayout>
  <UsersPage />
</DashboardLayout>
```

Все страницы получают единый каркас приложения.

---

### 3. Проверка прав доступа

Очень частый пример на собеседованиях.

```jsx
function PermissionGuard({ isAdmin, children }) {
  if (!isAdmin) {
    return <AccessDenied />;
  }

  return children;
}
```

Использование:

```jsx
<PermissionGuard isAdmin={user.isAdmin}>
  <AdminPanel />
</PermissionGuard>
```

Вложенный компонент будет отображен только при выполнении условия.

---

### 4. Защищенные маршруты

Похожий сценарий.

```jsx
function AuthGuard({ user, children }) {
  if (!user) {
    return <Navigate to="/login" />;
  }

  return children;
}
```

```jsx
<AuthGuard user={user}>
  <ProfilePage />
</AuthGuard>
```

Обертка инкапсулирует логику авторизации.

---

### 5. Обработка ошибок

На практике это часто реализуется через Error Boundary.

```jsx
<ErrorBoundary>
  <Dashboard />
</ErrorBoundary>
```

Если внутри произойдет ошибка рендеринга, приложение не упадет полностью.

---

### 6. Провайдеры контекста

Практически любой Provider является Wrapper Component.

```jsx
<AuthProvider>
  <App />
</AuthProvider>
```

или

```jsx
<ThemeProvider>
  <App />
</ThemeProvider>
```

Обертка предоставляет данные всему дереву компонентов.

---

### Wrapper Component и композиция

Компоненты-обертки являются одним из наиболее распространенных примеров композиции.

Например:

```jsx
<AuthGuard>
  <PageLayout>
    <Dashboard />
  </PageLayout>
</AuthGuard>
```

Каждая обертка добавляет свое поведение:

* `AuthGuard` проверяет авторизацию;
* `PageLayout` строит структуру страницы;
* `Dashboard` отвечает за бизнес-логику экрана.

Таким образом сложное поведение собирается из независимых компонентов.

---

### Wrapper Components и HOC

На собеседованиях часто спрашивают разницу между Wrapper Component и HOC.

Wrapper Component:

```jsx
<PermissionGuard>
  <AdminPanel />
</PermissionGuard>
```

HOC:

```jsx
const AdminPanelWithAuth =
  withAuth(AdminPanel);
```

Основное отличие:

* Wrapper Component работает через JSX и `children`;
* HOC принимает компонент и возвращает новый компонент.

После появления Hooks Wrapper Components используются значительно чаще.

---

### Wrapper Components и Hooks

Современный React обычно разделяет ответственность следующим образом:

* Hooks переиспользуют логику;
* Wrapper Components управляют структурой и поведением UI.

Например:

```jsx
function AuthGuard({ children }) {
  const { user } = useAuth();

  if (!user) {
    return <LoginPage />;
  }

  return children;
}
```

Логика получения пользователя находится в хуке, а обертка отвечает за отображение нужного интерфейса.
