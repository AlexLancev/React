### Как организовать код-сплиттинг на уровне маршрутов и тяжелых компонентов?

**Шпаргалка:**

Код-сплиттинг (Code Splitting) позволяет разбить большой JavaScript bundle на части и загружать их только тогда, когда они действительно нужны пользователю. Это уменьшает размер первоначальной загрузки и ускоряет старт приложения.

### Нюансы и выжимка для собеса

* код-сплиттинг уменьшает размер первоначального bundle
* реализуется через `React.lazy()` и `Suspense`
* чаще всего применяется на уровне маршрутов
* тяжёлые компоненты тоже можно загружать лениво
* хороший кандидат для lazy loading — код, который нужен не сразу
* не стоит дробить приложение на слишком маленькие чанки
* эффективность проверяют через bundle analyzer и DevTools
* цель код-сплиттинга — улучшить время первой загрузки приложения

---

### Своими словами

Код-сплиттинг — это способ не заставлять пользователя скачивать весь код приложения сразу. Обычно первым делом лениво загружают страницы маршрутов, потому что пользователь физически не может находиться на всех страницах одновременно. Дальше выделяют тяжёлые компоненты вроде графиков, карт или редакторов текста. Главная идея — загрузить только то, что нужно сейчас, а остальной код подтягивать по мере необходимости. Это уменьшает initial load и делает приложение быстрее при первом открытии.


---

#### Проблема большого bundle

Без код-сплиттинга:

```text
main.js
↓
весь код приложения
↓
загружается сразу
```

Даже если пользователь открыл только:

```text
/
```

он всё равно скачивает код для:

```text
/admin
/profile
/reports
/charts
```

---

#### Code Splitting через React.lazy

Базовый пример:

```jsx
import { lazy } from 'react';

const UserPage = lazy(() => import('./UserPage'));
```

React создаст отдельный chunk:

```text
UserPage.chunk.js
```

который загрузится только при необходимости.

---

#### Suspense обязателен

```jsx
import { Suspense, lazy } from 'react';

const UserPage = lazy(() => import('./UserPage'));

<Suspense fallback={<Loader />}>
  <UserPage />
</Suspense>
```

Пока chunk загружается:

```text
Loader
```

отображается пользователю.

---

#### Code Splitting на уровне маршрутов

Самый популярный сценарий.

React Router:

```jsx
const HomePage = lazy(() => import('./pages/Home'));
const ProfilePage = lazy(() => import('./pages/Profile'));
const AdminPage = lazy(() => import('./pages/Admin'));
```

```jsx
<Suspense fallback={<Loader />}>
  <Routes>
    <Route path="/" element={<HomePage />} />
    <Route path="/profile" element={<ProfilePage />} />
    <Route path="/admin" element={<AdminPage />} />
  </Routes>
</Suspense>
```

Получаем:

```text
home.chunk.js
profile.chunk.js
admin.chunk.js
```

#### Сплиттинг тяжёлых компонентов

Не только страницы могут быть ленивыми.

Например:

```text
Chart
Rich Text Editor
Map
PDF Viewer
```

Загружать их сразу часто бессмысленно.

---

Пример:

```jsx
const Chart = lazy(() => import('./Chart'));

function Dashboard() {
  return (
    <Suspense fallback={<Spinner />}>
      <Chart />
    </Suspense>
  );
}
```

---

#### Загрузка по условию

Например, модалка:

```jsx
{
  isOpen && <SettingsModal />;
}
```

Можно сделать:

```jsx
const SettingsModal = lazy(() => import('./SettingsModal'));
```

Код модалки загрузится только при открытии.

---

#### Prefetch и Preload

Если известно, что пользователь почти наверняка перейдёт дальше:

```text
Главная
↓
Каталог
```

можно заранее подгрузить chunk.

Идея:

```text
скачать заранее
↓
использовать позже
```

Это уменьшает задержку при переходе между страницами.

---

#### Как понять, что нужно выделять в отдельный chunk

Обычно кандидаты:

* страницы маршрутов
* админ-панели
* графики
* редакторы текста
* карты
* PDF/Excel просмотрщики
* редко используемые виджеты

---

#### Что не стоит лениво загружать

Мелкие компоненты:

```text
Button
Input
Avatar
Badge
```

Стоимость дополнительного запроса может быть больше выгоды.

---

#### Что смотреть в анализе bundle

Инструменты:

* webpack-bundle-analyzer
* Vite bundle visualizer
* Chrome DevTools Coverage

Помогают найти:

```text
самые тяжёлые зависимости
```

и вынести их в отдельные чанки.
