### Как определить, что компонент рендерится слишком часто?

**Шпаргалка:**

> Компонент рендерится слишком часто, если его повторные рендеры не приносят пользы UI, но при этом создают лишнюю нагрузку на приложение. Определяют это с помощью React DevTools Profiler, логирования рендеров и анализа причин обновления.

---

#### Простой способ — console.log

```jsx
function UserCard() {
  console.log('UserCard render');

  return <div>User</div>;
}
```

Если лог появляется чаще, чем ожидается, стоит разобраться в причинах.

Например:

* ввод в одном input вызывает ререндер всей страницы
* изменение одного поля формы ререндерит десятки компонентов
* клик по кнопке обновляет компоненты, которые не связаны с этим действием

---

#### Использование React DevTools Profiler

1. Открыть вкладку **Profiler**
2. Начать запись
3. Выполнить действие в приложении
4. Остановить запись
5. Посмотреть:

* какие компоненты ререндерились
* сколько раз рендерились
* сколько времени занял каждый рендер

Profiler — основной инструмент для поиска проблем производительности.

---

#### Признаки лишних ререндеров

Например:

```jsx
function Parent() {
  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(c => c + 1)}>
        +
      </button>

      <HeavyChart />
    </>
  );
}
```

Если `HeavyChart` не зависит от `count`, но рендерится при каждом клике — это кандидат на оптимизацию.

---

#### Полезный инструмент: Highlight Updates

В React DevTools можно включить:

```
Highlight updates when components render
```

React будет подсвечивать компоненты при каждом рендере.

Так можно быстро увидеть, какие части интерфейса обновляются слишком часто.

---

#### Когда частые рендеры — это нормально

Не каждый ререндер является проблемой.

Например:

```jsx
<input
  value={value}
  onChange={e => setValue(e.target.value)}
/>
```

Во время ввода ререндер на каждый символ — ожидаемое поведение.

Оптимизация нужна только тогда, когда:

* есть лаги
* падает FPS
* растёт время отклика UI
* Profiler показывает заметные затраты

---

### Нюансы и выжимка для собеса

* главный инструмент диагностики — React DevTools Profiler
* `console.log()` помогает быстро увидеть частоту рендеров
* Highlight Updates показывает визуально, какие компоненты обновляются
* не каждый ререндер является проблемой
* сначала ищем реальные узкие места, потом оптимизируем
* признак проблемы — компонент часто рендерится без изменения своих данных или влияния на UI

---

### Своими словами

Частый ререндер сам по себе не означает проблему. Сначала нужно понять, влияет ли он на производительность. Для этого используют React DevTools Profiler. Если компонент рендерится много раз, хотя его данные не меняются, или тяжёлый компонент обновляется из-за несвязанного state родителя, это сигнал к оптимизации. Главное правило — не оптимизировать вслепую, а сначала измерять.
