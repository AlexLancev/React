### Как оптимизировать большие списки (1000+ элементов)? Когда нужна виртуализация?

**Шпаргалка:**

Основная проблема больших списков — не сами данные, а количество DOM-узлов. Если React одновременно рендерит тысячи элементов, растут время рендера, потребление памяти и нагрузка на браузер. Для очень больших списков используют виртуализацию (windowing).

### Нюансы и выжимка для собеса

* проблема больших списков обычно связана с количеством DOM-узлов
* стабильные `key` помогают React эффективно выполнять reconciliation
* `React.memo` уменьшает лишние ререндеры строк списка
* фильтрацию и сортировку часто мемоизируют через `useMemo`
* после нескольких тысяч элементов основным узким местом становится DOM
* виртуализация рендерит только видимые элементы списка
* виртуализация значительно уменьшает память и стоимость рендера
* популярные решения: `react-window`, `TanStack Virtual`
* для небольших списков виртуализация обычно не нужна и только усложняет код

---

### Своими словами

Когда список становится большим, проблема чаще всего не в React, а в том, что браузеру приходится поддерживать тысячи DOM-элементов одновременно. Даже если данные не меняются, сам факт существования такого количества узлов начинает влиять на производительность. Поэтому сначала используют обычные оптимизации — правильные key, мемоизацию строк и вычислений. Если же элементов действительно много и список начинает тормозить при рендере или скролле, переходят к виртуализации. Её идея проста: пользователь видит только небольшую часть списка, значит нет смысла держать в DOM все элементы сразу. Это одна из самых эффективных оптимизаций для больших таблиц и списков.

---

#### Почему большие списки тормозят

Пример:

```jsx
{
  users.map(user => (
    <UserCard
      key={user.id}
      user={user}
    />
  ));
}
```

Если:

```text
users.length = 10000
```

React создаёт:

```text
10000 компонентов
↓
10000 DOM-узлов
↓
долгий render + reconciliation
↓
тяжёлый layout и paint
```

---

#### Базовые оптимизации

##### Стабильный key

Хорошо:

```jsx
key={user.id}
```

Плохо:

```jsx
key={index}
```

Стабильный ключ помогает React корректно сопоставлять элементы и уменьшает лишние обновления.

---

##### React.memo для элементов списка

```jsx
const UserRow = React.memo(({ user }) => {
  return <div>{user.name}</div>;
});
```

Если изменился один элемент:

```text
обновится только он
```

а не весь список.

---

##### Мемоизация вычислений

Если есть фильтрация:

```jsx
const filteredUsers = useMemo(() => {
  return users.filter(filterFn);
}, [users, filterFn]);
```

или сортировка:

```jsx
const sortedUsers = useMemo(() => {
  return sortUsers(users);
}, [users]);
```

---

#### Когда оптимизаций уже недостаточно

Даже если:

* правильные key
* React.memo
* useMemo

React всё равно может рендерить:

```text
5000+
10000+
50000+
элементов
```

Главная проблема остаётся:

```text
слишком много DOM
```

---

#### Что такое виртуализация

Идея:

```text
Показывать только видимую часть списка.
```

Например:

```text
10000 элементов всего

На экране помещается:
20 элементов

Рендерим:
20–40 элементов
```

При скролле старые элементы удаляются, новые добавляются.

---

#### Как это выглядит

Без виртуализации:

```text
10000 элементов
↓
10000 DOM-узлов
```

С виртуализацией:

```text
10000 элементов данных
↓
30 DOM-узлов
```

Размер данных не меняется, меняется количество отрисованных элементов.

---

#### Популярные библиотеки

Самые распространённые:

* [react-window](https://github.com/bvaughn/react-window?utm_source=chatgpt.com)
* [TanStack Virtual](https://tanstack.com/virtual?utm_source=chatgpt.com)

Пример:

```jsx
import { FixedSizeList } from 'react-window';

<List
  height={500}
  itemCount={10000}
  itemSize={40}
>
  {Row}
</List>
```

---

#### Когда виртуализация нужна

Практически всегда стоит рассмотреть её при:

```text
1000+
элементах
```

Особенно если:

* сложные карточки
* таблицы
* деревья
* бесконечный скролл
* тяжёлый UI внутри строки

---

#### Когда виртуализация не нужна

Если список:

```text
20
50
100
элементов
```

Обычный рендер обычно дешевле и проще.

Также виртуализация может усложнять:

* поиск по DOM
* анимации
* drag & drop
* автофокус
* измерение размеров элементов

---

#### Дополнительные техники

Пагинация:

```text
Показываем по 50 элементов
```

Infinite Scroll:

```text
Подгружаем данные частями
```

Ленивая загрузка тяжёлых компонентов:

```jsx
React.lazy(...)
```
