### В чем назначение useMemo и когда его имеет смысл использовать?

**Шпаргалка:**

`useMemo` — это хук для мемоизации значения. Он сохраняет результат вычисления между рендерами и пересчитывает его только тогда, когда изменяются зависимости.

### Нюансы и выжимка для собеса

* `useMemo` мемоизирует значение, а не функцию
* пересчитывает результат только при изменении зависимостей
* основные кейсы:

  * дорогие вычисления
  * стабилизация объектов и массивов
  * предотвращение лишних срабатываний `useEffect`
* помогает работать вместе с `React.memo`
* не стоит использовать для простых вычислений
* мемоизация тоже имеет стоимость
* применять после профилирования, а не заранее

---

### Своими словами

`useMemo` нужен для того, чтобы React не выполнял одно и то же вычисление на каждом рендере. Чаще всего его используют либо для тяжёлых вычислений, либо для создания стабильных объектов и массивов, которые передаются в дочерние компоненты или используются в зависимостях хуков. Если вычисление простое, например `count * 2`, то useMemo обычно не нужен, потому что стоимость мемоизации может оказаться выше самой операции.


---

#### Пример

```jsx
import { useMemo, useState } from 'react';

function App() {
  const [count, setCount] = useState(0);
  const [theme, setTheme] = useState('light');

  const expensiveValue = useMemo(() => {
    console.log('expensive calculation');

    return count * 1000;
  }, [count]);

  return (
    <>
      <div>{expensiveValue}</div>

      <button onClick={() => setCount(c => c + 1)}>
        Count
      </button>

      <button onClick={() => setTheme('dark')}>
        Theme
      </button>
    </>
  );
}
```

При изменении `theme`:

```jsx
setTheme('dark');
```

компонент ререндерится, но вычисление внутри `useMemo` не запускается, потому что `count` не изменился.

---

#### Как это работает

Без `useMemo`:

```jsx
const result = expensiveCalculation(data);
```

функция будет вызываться на каждом рендере.

С `useMemo`:

```jsx
const result = useMemo(() => {
  return expensiveCalculation(data);
}, [data]);
```

React:

1. сохраняет результат вычисления
2. сравнивает зависимости
3. если зависимости не изменились — возвращает сохранённое значение
4. если изменились — пересчитывает результат

---

#### Частый кейс №1 — дорогие вычисления

```jsx
const filteredUsers = useMemo(() => {
  return users.filter(user => user.active);
}, [users]);
```

Полезно когда:

* большие массивы
* сложная сортировка
* агрегации данных
* тяжёлые вычисления

---

#### Частый кейс №2 — стабильные объекты

Без мемоизации:

```jsx
const options = {
  theme: 'dark'
};
```

На каждом рендере создаётся новый объект.

С мемоизацией:

```jsx
const options = useMemo(() => {
  return {
    theme: 'dark'
  };
}, []);
```

Ссылка остаётся стабильной.

Это важно для:

```jsx
<Child options={options} />
```

если `Child` использует:

```jsx
React.memo()
```

---

#### Частый кейс №3 — зависимости useEffect

Без мемоизации:

```jsx
const config = {
  page: 1
};

useEffect(() => {
  fetchData(config);
}, [config]);
```

Эффект будет запускаться на каждом рендере.

С `useMemo`:

```jsx
const config = useMemo(() => {
  return {
    page: 1
  };
}, []);
```

ссылка остаётся неизменной.

---

#### Когда useMemo не нужен

Плохой пример:

```jsx
const doubled = useMemo(() => count * 2, [count]);
```

Вычисление слишком дешёвое.

Накладные расходы на мемоизацию могут быть выше пользы.
