### Как изменение состояния в родительском компоненте влияет на дочерние компоненты?

**Шпаргалка:**

> Когда состояние (state) родительского компонента изменяется, React повторно вызывает функцию родительского компонента. По умолчанию вместе с ним будут повторно вызваны и все его дочерние компоненты.

---

#### Пример

```jsx
function Child() {
  console.log('Child render');

  return <div>Child</div>;
}

function Parent() {
  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(c => c + 1)}>
        {count}
      </button>

      <Child />
    </>
  );
}
```

При каждом нажатии кнопки:

```jsx
setCount(c => c + 1);
```

происходит:

1. обновление state
2. ререндер `Parent`
3. повторный вызов `Child`

Даже если `Child` не использует `count`.

---

#### Почему так происходит

React не анализирует содержимое компонента заранее.

Когда родитель рендерится заново:

```jsx
<Child />
```

создаётся новый React Element.

Поэтому React вызывает дочерний компонент и проверяет, изменился ли результат его рендера.

---

#### Как избежать лишних ререндеров

Использовать `React.memo`.

```jsx
const Child = React.memo(function Child() {
  console.log('Child render');

  return <div>Child</div>;
});
```

Теперь `Child` будет ререндериться только если изменятся его props.

---

#### Когда React.memo не поможет

Если в props передаются новые объекты или функции:

```jsx
<Child options={{ theme: 'dark' }} />
```

или

```jsx
<Child onClick={() => console.log('click')} />
```

На каждом рендере создаются новые ссылки.

Для таких случаев используют:

```jsx
useMemo()
useCallback()
```

---

#### Влияние на большое дерево компонентов

Если родитель находится высоко в дереве:

```jsx
<App>
  <Layout>
    <Sidebar />
    <Content />
    <Footer />
  </Layout>
</App>
```

то изменение его state потенциально запускает ререндер большого количества компонентов.

Поэтому важно:

* хранить state как можно ближе к месту использования
* разбивать UI на небольшие компоненты
* использовать `React.memo` там, где это действительно полезно

---

### Нюансы и выжимка для собеса

* изменение state родителя вызывает его ререндер
* по умолчанию ререндерятся и все дочерние компоненты
* дочерний компонент может ререндериться даже если его props не изменились
* `React.memo` позволяет пропускать лишние ререндеры по props
* новые объекты и функции ломают мемоизацию из-за новых ссылок
* для стабилизации ссылок используют `useMemo` и `useCallback`
* хранить state стоит максимально близко к месту использования

---

### Своими словами

Когда меняется state родителя, React заново запускает его функцию. Поскольку внутри родителя заново создаётся дерево JSX, дочерние компоненты тоже по умолчанию вызываются повторно. Это не всегда проблема, потому что React эффективно обновляет DOM. Но если дочерние компоненты тяжёлые или их много, могут появиться лишние затраты на рендеринг. В таких случаях помогают `React.memo`, `useMemo` и `useCallback`, а также грамотное размещение состояния в дереве компонентов.
