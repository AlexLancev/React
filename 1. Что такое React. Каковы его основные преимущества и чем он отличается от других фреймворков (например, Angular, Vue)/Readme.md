# React: Основные концепции

React — это популярная JavaScript-библиотека для построения интерфейсов. Её главные преимущества: компонентная архитектура с JSX, высокая производительность за счёт Virtual DOM и однонаправленный поток данных. В отличие от полноценных фреймворков (Angular), React фокусируется только на слое представления, оставляя выбор остальных инструментов (роутинг, менеджер состояния) за разработчиком, что обеспечивает гибкость. Vue предлагает похожий компонентный подход, но с более низким порогом входа и встроенными директивами.

## Ключевые нюансы React

### 1. Библиотека, а не фреймворк
React отвечает только за представление (View), что дает свободу в выборе остальных инструментов, но требует больше решений от разработчика.

### 2. Декларативность и однонаправленный поток
Вы описываете *что* должно отобразиться (в зависимости от состояния), а не *как* это сделать (императивный подход). Данные явно передаются от родителей к детям, что упрощает отслеживание изменений.

### 3. Экосистема vs. "все из коробки"
Главное отличие от Angular. React предоставляет гибкость, Angular — строгую структуру "под ключ". Vue находится посередине.

## Сравнение с другими технологиями

### React vs. Angular
- **React**: Библиотека, фокус на представлении, гибкость в выборе инструментов
- **Angular**: Полноценный фреймворк, строгая структура "под ключ"

### React vs. Vue
- **React**: Больше контроля, требует дополнительных решений
- **Vue**: Более низкий порог входа, встроенные директивы

## Простыми словами

> React — это инструмент для сборки интерфейса из готовых блоков-компонентов. 
> 
> Вместо того чтобы вручную обновлять страницу при каждом изменении данных, ты просто описываешь, как должен выглядеть каждый блок в разных состояниях, а React сам эффективно всё перерисовывает. 
> 
> В отличие от больших фреймворков вроде Angular, React решает только задачу отрисовки, поэтому его можно гибко комбинировать с другими библиотеками.
